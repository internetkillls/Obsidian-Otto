# Otto-Realm Maintenance Patterns

## Purpose

Keep Otto-Realm small, durable, and useful as a private continuity layer for Otto.

## Note roles

### Profile Snapshot
Use for:
- stable identity traits
- durable manner guidelines
- repeated strengths or weaknesses
- long-lived motifs

Do not use for:
- temporary bugs
- one-off frustrations
- detailed session logs

### Heartbeats
Use for:
- recent interaction quality
- friction signals
- follow-through audit
- next repair targets
- short-term telemetry

Do not use for:
- large strategy essays
- permanent identity claims without repetition

### Weekly notes
Use for:
- current overhaul themes
- multi-day priorities
- active experiments
- cross-session repair plans

### Central Schedule
Use for:
- real reminders or shared timing anchors only

## Update heuristics

### Write to Profile Snapshot when
- the pattern has repeated enough to feel durable
- it changes how Otto should speak or work
- it should survive across many chats

### Write to Heartbeat when
- the signal is recent
- the quality of interaction changed
- Otto missed, drifted, stalled, or recovered
- there is a concrete repair to test next

### Write to Weekly when
- several heartbeat items form one broader theme
- Otto needs an overhaul target for multiple days
- the issue affects channel parity, retrieval discipline, or behavior design

## Preferred structure for updates

1. observation
2. why it matters
3. repair or reinforcement
4. where it should persist

## Telegram-specific repair prompts

Use these kinds of repair targets when Telegram quality is weak:
- restore self-context before answering relational prompts
- send progress updates during longer work
- never claim background execution unless it has started
- finish with deliverable or blocker
- convert awkward moments into heartbeat notes

## Interaction quality lens

Look for:
- warmth without vagueness
- structure without stiffness
- initiative without pretending work was already done
- continuity without inventing memory
